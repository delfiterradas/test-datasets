# SAVANA ONT CNA test data

Minimal ONT BAM files and a chromosome 22 FASTA reference for testing SAVANA.

- **Source**: [SAVANA issue 111](https://github.com/cortes-ciriano-lab/savana/issues/111), provided by the @helrick & @cmsauer as test files
- **Original data**: ONT EPI2ME public sequencing data from the COLO829 melanoma cell line and matched normal cell line COLO829BL
- **Region**: BAMs were subset to `chr22:24000000-25000000`
- **Additional downsampling**: The test BAMs provided [here](https://github.com/cortes-ciriano-lab/savana/issues/111) were downsampled with `samtools view -s 0.05`
- **Reference**: `hg38.chr22.fasta`, hg38 subset to chromosome 22
- **Purpose**: Small fixture data for testing that SAVANA can run and produce output files

These data are not sufficient for actual copy number calling. SAVANA is expected to report `unable to fit solution`, but should still produce output files and exit with status 0 when run with the additional parameters.

### Test files

| File | Size | Description |

| `hg38.chr22.fasta.fai` | 27 B | FASTA index |

> [!NOTE]
> The fasta and its index is stored under `data/genomics/homo_sapiens/genome/chr22/`

## How they were generated

@helrick & @cmsauer generated a subset of publicly available ONT EPI2ME sequencing of the COLO829 melanoma cell line and its matched normal cell line COLO829BL. The normal and tumour BAMs were subset to `chr22:24000000-25000000`.

The reference FASTA was generated by subsetting hg38 to chromosome 22. This replaced an earlier incomplete chromosome 22 FASTA used in the upstream issue, which caused errors.

The smaller `.0.05.bam` files in this directory were generated from the upstream chr22 subset BAMs with `samtools view --write-index --no-PG -b -s 0.05`.
